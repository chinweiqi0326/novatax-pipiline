# Nova Tax SG - Sales Pipeline 部署指南

## 总共3步，15分钟搞定

---

## 第1步：Supabase（数据库）

1. 打开 https://supabase.com → 注册/登录（用GitHub登录最快）
2. 点 "New Project" → 取名 "novatax-pipeline" → 设密码 → Region选 Singapore → 点 Create
3. 等1-2分钟project建好
4. 左边菜单点 "SQL Editor" → 点 "New Query"
5. 把 `supabase_setup.sql` 文件里的内容全部复制粘贴进去 → 点 "Run"
6. 左边菜单点 "Project Settings" → "API" → 复制这两个值：
   - Project URL（类似 https://xxxxx.supabase.co）
   - anon public key（很长的一串）

---

## 第2步：Vercel（部署）

1. 把整个 novatax-pipeline 文件夹上传到 GitHub（新建一个repo）
2. 打开 https://vercel.com → 用GitHub登录
3. 点 "Add New Project" → 选你刚才的repo → 点 Import
4. 在 "Environment Variables" 里加两个：
   - `NEXT_PUBLIC_SUPABASE_URL` = 你的Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = 你的anon key
5. 点 "Deploy" → 等1-2分钟
6. 部署完成！你会拿到一个链接（类似 https://novatax-pipeline.vercel.app）

---

## 第3步：使用

1. 你和Sam都用手机/电脑打开那个链接
2. 你添加的lead，Sam马上能看到（实时同步）
3. Sam更新了状态，你也马上看到
4. 建议：把链接添加到手机主屏幕，像app一样打开

---

```

├── package.json
├── next.config.js
├── .env.local.example    ← 复制成 .env.local 填上你的key（本地开发用）
├── supabase_setup.sql    ← 在Supabase SQL Editor里跑
└── src/
    ├── lib/
    │   └── supabase.js
    └── app/
        ├── layout.js
        └── page.js
```

---

## 如果遇到问题

- "连接失败" → 检查Supabase的URL和Key有没有填对
- 数据不显示 → 检查有没有跑supabase_setup.sql
- 部署失败 → 检查Vercel的Environment Variables

有问题找老婆！
